🛠 Level Editor Integration

This WebAssembly build supports direct level boot via URL parameters and a full terrain override API for custom level testing workflows.

📱 Android

Native Android builds are available in the repository. The WebAssembly build is recommended for quick testing and level preview.

🔗 Skip-to-Level URL Parameters

Append these query parameters to skip title screens and menus for rapid level testing:

?level=N — Boot directly into level N (0–9), skipping all title screens and menus.

?terrain=/Data/Terrain/custom.ter — Load a custom terrain file for the level.

OttoMatic.html?level=1
OttoMatic.html?level=3
OttoMatic.html?level=0&terrain=/Data/Terrain/MyLevel.ter

🧪 JavaScript API (browser console)

Once the game is loaded, use the PangeaGame object or direct Module calls from the browser console:

// Standard API (available after game loads)
PangeaGame.skipToLevel(3);              // skip to level 3 (0–9)
PangeaGame.getCurrentLevel();           // returns current level number

// Game-specific terrain override
PangeaGame.setTerrainPath('/Data/Terrain/custom.ter');

// Direct C function equivalents
OttoMatic_SkipToLevel(3);
OttoMatic_GetCurrentLevel();
OttoMatic_SetTerrainPath('/Data/Terrain/custom.ter');
