  • Arrow keys / WASD — move
  • Space — jump / boost
  • Ctrl / Left-click — fire weapon
  • Tab — switch weapon
  • Escape — pause
  • Click canvas to capture mouse

🛠 Level Editor Integration

This WebAssembly build supports direct level boot via URL parameters, making it easy to iterate on custom terrain files without navigating through the title screen every time.

📱 Android

Native Android builds are available in the repository. The WebAssembly build is recommended for quick testing and level preview.

🔗 Skip-to-Level URL Parameters

Append these query parameters to skip title screens and menus for rapid level testing:

?level=N — Boot directly into level N (0–8), skipping all title screens and menus.

?terrainOverride=:Terrain:custom.ter — Load a custom terrain file instead of the built-in one.

Nanosaur2.html?level=0
Nanosaur2.html?level=3
Nanosaur2.html?level=0&terrainOverride=:Terrain:MyLevel.ter

🧪 JavaScript API (browser console)

Once the game is loaded, use the PangeaGame object or direct Module calls from the browser console:

// Standard API (available after game loads)
PangeaGame.skipToLevel(3);              // skip to level 3
PangeaGame.getCurrentLevel();           // returns current level number

// Direct Module calls (equivalent)
Module.ccall('Nanosaur2_GetCurrentLevel', 'number');
Module.ccall('Nanosaur2_SetFenceCollisionsEnabled', null, ['number'], [0]);
